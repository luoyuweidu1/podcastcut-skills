# 静音段处理（播客版）

## 阈值规则（播客专用）

| 静音长度 | 处理 | 说明 |
|---------|------|------|
| ≤ 1秒 | **忽略** - 自然停顿 | 播客需要呼吸空间 |
| 1-2秒 | **审慎处理** - 可能是思考停顿 | 保留大部分，除非明显异常 |
| 2-5秒 | **建议删除** - 明显卡顿 | 根据上下文判断 |
| > 5秒 | **必须删除** - 严重问题 | 剪辑事故或技术问题 |

## 播客特殊考虑

### 多人对话
- 对话轮换之间的停顿（1-1.5秒）是正常的，**不要删除**
- 问题提出后的思考时间（1-2秒）**应该保留**
- 只有异常长的停顿才需要处理

### 单人播客
- 可以比多人对话更紧凑
- 但思考停顿（1-1.5秒）仍应适度保留
- 2秒以上才考虑删除

## 输出格式

**整段标记，不拆分**

示例：3.2 秒静音 → 输出 1 条
```
| 64-66 | 12.86-15.80 | 静音3.2s | 超过2秒阈值 | 删 |
```

示例：1.5 秒静音 → 通常保留
```
| 64-66 | 12.86-14.36 | 静音1.5s | 自然停顿 | 保留 |
```

用户在审核网页中可以调整决策。

## 特殊情况

### 长静音
连续 5s+ 的静音整段标记，预选删除：
```
| 323-371 | 71.38-131.38 | 静音60s | 严重异常 | 删 |
```

### 开头/结尾静音
- 播客开头的静音（>1秒）建议删除
- 结尾静音看情况，可能是淡出效果

### 上下文判断
如果静音前后是同一话题的连续表达，即使稍长（1.5-2秒）也可以保留。
如果静音看起来是剪辑事故或技术问题，即使短于2秒也应删除。
